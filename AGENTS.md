# Apple Pen Reader 作成要領

このリポジトリは、iPad のブラウザで PDF を開き、Apple Pencil・タッチ・マウスで囲った文字を抽出する HTML プロトタイプを扱います。
このドキュメントは、今後このコードを編集する人向けの作業ルールです。

## 目的

- PDF を iPad ブラウザ上で表示する
- Apple Pencil・タッチ・マウスで文字を囲めるようにする
- 囲まれた範囲内の PDF テキストを抽出する
- まずは試作を優先し、複雑な最適化は後回しにする

## 既定の構成

- `index.html` にアプリ本体を置く
- `README.md` に利用手順と実装メモを置く
- 変更は必要最小限に保ち、関連するファイルだけを触る

## 実装方針

- UI は標準の HTML/CSS/JavaScript を基本にする
- PDF 表示とテキスト取得は PDF.js を使う
- 描画入力は Pointer Events と Canvas を使う
- 文字抽出は PDF.js の text content と表示座標を基準に考える
- ペンで描いた線の座標変換は、表示サイズと Canvas 座標の差を意識して実装する
- まずは単一ページの動作を安定させ、その後に複数ページ対応を広げる

## 変更時の注意

- 既存の挙動を壊す変更は避ける
- 文字コードやファイル形式は、特別な理由がなければそのまま維持する
- 大きい変更は、一度に詰め込みすぎず、動作確認しやすい単位で分ける
- 画面上の文言は、iPad での見やすさを優先する

## 確認項目

- PDF を開けるか
- Apple Pencil・タッチ・マウスで描画できるか
- 囲んだ範囲から文字を抽出できるか
- 画面回転やサイズ変更で表示が崩れないか

## 進め方

1. 変更前に対象ファイルを確認する
2. 実装は小さくまとめる
3. 変更後にビルドや動作確認を行う
4. README やこの要領が古くなったら同時に更新する

## コミットの考え方

- 1 コミット 1 目的を基本にする
- コミットメッセージの先頭には、必ず `V001` のように `V` と3桁の連番で構成したバージョンナンバーを付ける
- `index.html` の画面左上には、現在のバージョンナンバーを表示する
- PR には、何を変えたかと何を確認したかを簡潔に書く
- 試作段階でも、あとから追える説明を残す
